# Contributing Guidelines

Grazie per voler contribuire a **stageMatch**. Questo documento spiega le regole base per collaborare al progetto in modo ordinato

## Indice

- [Collaboratori](#collaboratori)
- [Contributori](#contributori)

---

## Collaboratori

### 1. Creazione del branch

Ogni collaboratore deve creare un **proprio branch partendo da `main`**, nominato con il **proprio nome** (prima lettera maiuscola)

Esempi:
```bash
git checkout main
git pull origin main
git checkout -b Falcone

git checkout main
git pull origin main
git checkout -b Rossi
```

---

### 2. Messaggi nei commit

Ogni commit deve iniziare con il **proprio nome in maiuscolo**, seguito da due punti e una breve descrizione

Esempi:
```
FALCONE: fix login redirect

ROSSI: update frontend layout
```

➡️ Evita messaggi generici come "update files" o "changes"

---

### 3. Pull Request

Quando il lavoro è pronto:

1. Fai `git push` sul tuo branch
2. Apri una **Pull Request (PR)** verso `main`
3. Nella PR, descrivi chiaramente cosa hai modificato o aggiunto

Ogni PR sarà revisionata dal team prima di essere accettata

## Contributori

### 1. Creazione del branch

Ogni contributore deve creare un **proprio branch partendo da `dev`**, nominato con il **proprio nome** (prima lettera maiuscola)

Esempi:
```bash
git checkout dev
git pull origin dev
git checkout -b Contributore

git checkout dev
git pull origin dev
git checkout -b Sviluppatore
```

---

### 2. Messaggi nei commit

Ogni commit deve iniziare con il **proprio nome in maiuscolo**, seguito da due punti e una breve descrizione

Esempi:
```
CONTRIBUTORE: add new feature

SVILUPPATORE: update documentation
```

➡️ Evita messaggi generici come "update files" o "changes"

---

### 3. Pull Request

Quando il lavoro è pronto:

1. Fai `git push` sul tuo branch
2. Apri una **Pull Request (PR)** verso `dev`
3. Nella PR, descrivi chiaramente cosa hai modificato o aggiunto

Ogni PR sarà revisionata dal team prima di essere accettata

---

### 4. Segnalazione bug

Se trovi un problema:

- Crea una **Issue** nel repository
- Includi una descrizione chiara e, se possibile, come riprodurlo
